THEORETICAL QUESTIONS 1. Explain the main differences between fixed-point and floating-point number representations. Fixed-point: Decimal point at fixed position, limited range but faster processing Floating-point: Decimal point "floats" using exponent, wider range but slower 2. Convert -5.75 to its binary form in IEEE 754 single-precision format. -5.75 = -101.11 binary = -1.0111 × 2² Sign: 1 (negative) Exponent: 2 + 127 = 129 = 10000001 Mantissa: 01110000000000000000000 Final: 1 10000001 01110000000000000000000 3. Show the 8-bit two’s complement binary representation of -25. 25 = 00011001 Invert: 11100110 Add 1: 11100111 4. Explain the difference between direct and complement codes with an example. Direct code: first bit shows sign, the rest shows the value Complement code: Invert bits then add 1for negative 5. Describe how to convert a given two’s complement code back to decimal form. If first bit is 0, read normally If first bit is 1, invert bits, add 1 and negative sign 6. Specify the range of signed numbers that can be represented with 1 byte (8 bits). -128 127 7. Explain how overflow can be detected when adding two 16-bit numbers. Adding two positives gives a negative result or adding two negatives give a positive result. If carry into and carry out of sign bit is different, there is overflow 8. Describe the roles of mantissa and exponent in a floating-point number. Mantissa: holds main digits Exponent: how many times to move decimal point 1.23 * 103 mantissa = 1.23 exponent = 3 9. What is the normalization of a fixed-point number and why is it important? Adjusting the number so there is only one non-zero digit before decimal point. Important for accuracy and efficient use of bits. 10. Define the concept of bias in IEEE 754 format and calculate the bias value for an 8-bit exponent field. Bias makes the exponent positive. Formula: 2n-1 -1 28-1 -1 = 127 11. List the main parts of the CPU and briefly describe the function of each. Control Unit: controls all operations Arithmetic Logic Unit: does calculations and logic operations Registers: store small, fast data 12. What is the function of the ALU, and what types of operations does it perform? Arithmetic operations (add, subtract, multiply, divide), Logic operations (AND, OR, NOT, compare) 13. Explain the different roles of the Program Counter (PC) and the Stack Pointer (SP) registers. PC: Holds the address of the next instruction to run. SP: Points to the top of the stack 14. Differentiate between general-purpose and special-purpose registers. General: can be used for various purposes (data, temporary storage) Special: used by CPU 15. Describe the Accumulator-based machine model and give an example. All operations use one main register called the Accumulator. To add two numbers, load first into Accumulator, add the second, result stays in Accumulator. 16. Compare stack-based and register-based CPU architectures. Stack: operations work on stack top, data stored in stack, slower Register: work and store data in multiple registers, faster 17. Describe the phases of the Instruction Cycle (Fetch, Decode, Execute). Fetch: get instruction from memory Decode: understand what to do Execute: perform operation 18. List the main characteristics of a microprocessor. clock speed, instruction set, word length, cache memory, and bus speed 21. Describe the levels of cache memory (L1, L2, L3) and their role between the CPU and RAM. L1: Fastest, smallest, inside CPU core L2: Larger than L1, slower, may be shared L3: Largest cache, shared between cores 22. List and explain the main differences between RAM and ROM. RAM: Volatile, read/write, temporary storage ROM: Non-volatile, read-only, permanent storage 23. Describe the working principle of a Stack (LIFO) and the instructions used to manage it. Last In First Out Push to add data to top, Pop to remove data from top 24. Compare DRAM and SRAM in terms of speed, cost, and structure. DRAM: Cheaper, slower, needs refresh, used for main memory SRAM: More expensive, faster, no refresh, used for cache 25. Explain the difference between address bus and data bus. Address bus: Sends the address of memory location. (One-way) Data bus: Transfers actual data. (Two-way) 26. What are cache hit and cache miss events in CPU operation? Hit: Data found in cache (fast access) Miss: Data not in cache, must get from main memory (slow) 27. Describe the memory hierarchy model and arrange its levels in order of access speed. Registers → L1 Cache → L2 Cache → L3 Cache → RAM → Storage 31. List the main components on a motherboard and describe the function of each. CPU socket – Holds the processor RAM slots – For memory Chipset – Connects CPU and devices BIOS/UEFI chip – Starts the computer Power connector, Ports, Expansion slots 32. Explain the three main types of buses: data, address, and control. Data bus: Transfers data Address bus: Specifies memory locations Control bus: Carries control signals 33. Differentiate between serial and parallel ports. Serial: One bit at a time (USB, Ethernet) Parallel: Multiple bits simultaneously (older printers) 34. Explain the key difference between PCI and USB interfaces. PCI: Internal expansion bus for cards USB: External connection for peripherals 35. List the main differences between Von Neumann and Harvard architectures. Von Neumann: Single memory for data and instructions, slower Harvard: Separate memories for data and instructions, faster 36. What does a processor’s Instruction Set Architecture (ISA) define? Instruction formats, data types, registers, addressing modes 37. Describe the different addressing modes (Immediate, Direct, Indirect, Indexed, Register). Immediate: Data in instruction itself Direct: Instruction contains address Indirect: Instruction points to address of address Indexed: Address = base + index register Register: Data in register 38. What is the function of the Control Unit (CU), and how does it interact with other CPU components?
