THEORETICAL QUESTIONS & ANSWERS 1. Explain the main differences between fixed-point and floating-point number representations. • Fixed-point: binary point fixed; numbers have uniform precision, range limited by word size; arithmetic is simple and fast; often used in embedded systems, digital signal processing, or situations where predictable results are required. • Floating-point: numbers represented as sign × mantissa × 2^exponent; allows very large and very small numbers; precision depends on magnitude (significant digits stored in mantissa); hardware more complex (normalization, rounding, handling special cases like NaN/Infinity). • Key difference: fixed-point has limited range but consistent precision; floating-point supports wide range with variable precision. 2. Convert -5.75 to its binary form in IEEE 754 single-precision format. • Convert absolute value: 5.75 = 101.11 in binary. • Normalize: 1.01111 × 2^2 (shift binary point so one digit before point). • Sign bit = 1 (negative). • Exponent = 2 + 127 = 129 → 10000001. • Mantissa = fractional bits after leading 1: 01111000000000000000000. • Result (32-bit): 1 10000001 01111000000000000000000. • IEEE 754 allows standardized storage of signed fractional numbers for calculation across systems. 3. Show the 8-bit two’s complement binary representation of -25. • +25 in binary = 00011001 • Invert all bits = 11100110 • Add 1 → 11100111 • Explanation: two’s complement allows subtraction by addition; negative numbers integrate seamlessly into arithmetic operations without separate logic. 4. Explain the difference between direct and complement codes with an example. • Direct (sign-magnitude): MSB = sign, remaining bits = magnitude; +6 = 00000110, −6 = 10000110. • Two’s complement: negative = invert + 1; +6 = 00000110, −6 = 11111010. • Reason for preference: two’s complement avoids dual zeros, simplifies arithmetic (addition/subtraction), and is standard in modern CPUs. 5. Describe how to convert a given two’s complement code back to decimal form. • Check MSB: 0 → positive → convert normally; 1 → negative → invert bits + 1 → convert → apply minus. • Example: 11100111 → invert 00011000 → add 1 → 00011001 = 25 → −25. • Ensures signed numbers are correctly interpreted; crucial for arithmetic, memory storage, and instruction decoding. 6. Specify the range of signed numbers that can be represented with 1 byte (8 bits). • Two’s complement 8-bit: −2^7 to 2^7−1 → −128 to 127. • MSB = sign, remaining 7 bits = magnitude. • Explains why CPU flags overflow when operations exceed these limits. 7. Explain how overflow can be detected when adding two 16-bit numbers. • Unsigned overflow: carry out of MSB → CF = 1; indicates result exceeds max representable unsigned value. • Signed overflow: occurs when adding two positives yields negative, or two negatives yield positive → OF = 1. • Detection method: compare carry into MSB with carry out; flags ensure arithmetic integrity and prevent incorrect results. 8. Describe the roles of mantissa and exponent in a floating-point number. • Mantissa: stores significant digits, determines precision; e.g., 1.01111 in binary. • Exponent: scales mantissa by 2^n, determines the numeric range. • Combined role: allows representation of very large/small numbers efficiently; floating-point arithmetic uses normalization, rounding, and bias to maintain accuracy. 9. What is the normalization of a fixed-point number and why is it important? • Normalization: shift number so most significant bit is 1 (for fixed-point or floating-point). • Importance: maximizes precision, ensures a unique representation, simplifies arithmetic operations, and reduces rounding errors. • Example: 00101.10 → normalized 1.01011 × 2^2. 10. Define the concept of bias in IEEE 754 format and calculate the bias value for an 8-bit exponent field. • Bias: fixed value added to actual exponent to store it as unsigned, allowing positive and negative exponents. • Formula: bias = 2^(k−1) − 1 • Example (8-bit exponent): 2^7 − 1 = 127 • Bias simplifies comparison, arithmetic, and storage of floating-point numbers; ensures exponents are interpreted correctly. 11. List the main parts of the CPU and briefly describe the function of each. • ALU (Arithmetic Logic Unit): performs arithmetic (ADD, SUB, MUL, DIV) and logic operations (AND, OR, XOR, NOT), and updates status flags. • Control Unit (CU): fetches instructions from memory, decodes them, and generates control signals to coordinate ALU, registers, memory, and I/O devices. • Registers: small, fast storage inside CPU for temporary data, addresses, and intermediate results. Examples: general-purpose (AX, BX) and special-purpose (PC, SP, FLAGS). • Memory Interface / Cache: stores frequently used data and instructions for fast access. • Buses: data, address, and control lines connecting CPU, memory, and peripherals; enable data transfer. 12. What is the function of the ALU, and what types of operations does it perform? • Function: performs all arithmetic and logic operations, enabling CPU to calculate, compare, and make decisions. • Arithmetic operations: ADD, SUB, INC, DEC, MUL, DIV. • Logic operations: AND, OR, XOR, NOT. • Comparison operations: CMP (sets flags for conditional instructions). • ALU interacts with registers and memory, and results update flags in the status register for program control. 13. Explain the different roles of the Program Counter (PC) and the Stack Pointer (SP) registers. • PC (Program Counter): holds the memory address of the next instruction to execute; increments automatically after fetching; controls sequential program flow. • SP (Stack Pointer): points to the top of the stack in memory; used for PUSH/POP operations, saving registers, managing function calls, and local variables. • Summary: PC manages instruction sequencing; SP manages temporary storage and subroutine execution. 14. Differentiate between general-purpose and special-purpose registers. • General-purpose registers: store temporary data for arithmetic, logic, and data movement (e.g., AX, BX, CX, DX). Flexible use. • Special-purpose registers: dedicated to specific CPU operations, e.g., PC (next instruction), SP (stack top), FLAGS (status). • General-purpose registers are versatile; special-purpose registers control program execution and CPU status. 15. Describe the Accumulator-based machine model and give an example. • Model: CPU uses a single main register (accumulator) for all arithmetic and logic operations. • Operation: one operand comes from accumulator, other from memory or register; result stored in accumulator. • Example: ADD AX, BX → AX = AX + BX. • Advantage: simple CPU design. Disadvantage: may require more memory access for multiple operands. 16. Compare stack-based and register-based CPU architectures. • Stack-based CPU: operations use top of stack (PUSH/POP); fewer registers required; instruction encoding simpler; memory access slower. • Register-based CPU: operations use explicit registers; faster execution; more registers needed; instructions specify source/destination operands. • Summary: Stack-based good for compiler-friendly designs; register-based faster for general-purpose computation. 17. Describe the phases of the Instruction Cycle (Fetch, Decode, Execute). 1. Fetch: CPU reads instruction from memory; PC points to instruction; instruction stored in instruction register (IR). 2. Decode: CU interprets opcode, identifies operation, source, and destination operands. 3. Execute: ALU performs arithmetic/logic operation; results stored in register or memory; flags updated. • Importance: repeated for each instruction; forms the core operation cycle of all CPUs. 18. List the main characteristics of a microprocessor. • CPU integrated on a single chip; contains ALU, CU, registers. • Executes instructions sequentially; programmable. • Interfaces with memory (RAM/ROM) and I/O devices. • Performs arithmetic, logic, control, and data transfer operations. • Compact, fast, and used in computers, embedded systems, and controllers. • Supports interrupts, timers, and sometimes built-in cache for performance. 21. Describe the levels of cache memory (L1, L2, L3) and their role between the CPU and RAM. • L1 cache: smallest, fastest, located inside CPU core; stores frequently used instructions/data for immediate access. • L2 cache: larger, slightly slower; may be inside CPU or on-chip; stores additional frequently used data. • L3 cache: largest, slowest, often shared between CPU cores; reduces access to main memory. • Role: acts as an intermediary between CPU and RAM, reducing latency and speeding up execution by keeping frequently used data closer to the processor. 22. List and explain the main differences between RAM and ROM. • RAM (Random Access Memory): volatile, read/write, stores program data temporarily; contents lost when power is off; used by CPU for running programs. • ROM (Read-Only Memory): non-volatile, typically read-only, stores firmware or boot instructions; data persists without power. • Key differences: volatility, mutability, and purpose. RAM is for dynamic operation; ROM is for permanent storage. 23. Describe the working principle of a Stack (LIFO) and the instructions used to manage it. • Principle: Last-In, First-Out; the last item pushed is the first item popped. • Instructions: PUSH (adds data to top of stack), POP (removes data from top). • Usage: storing return addresses, local variables, saving registers during function calls; essential for subroutine management and interrupt handling. 24. Compare DRAM and SRAM in terms of speed, cost, and structure. • DRAM (Dynamic RAM): stores bits in capacitors, requires refresh cycles; slower, cheaper; used as main memory. • SRAM (Static RAM): stores bits in flip-flops; faster, more expensive; no refresh needed; used for CPU caches. • Summary: DRAM = large, inexpensive, slower; SRAM = small, fast, costly. 25. Explain the difference between address bus and data bus. • Address bus: carries memory addresses from CPU to RAM/I-O to indicate where data should be read/written. • Data bus: carries actual data between CPU, memory, and I/O devices. • Summary: address bus specifies location (“where”); data bus carries content (“what”). 26. What are cache hit and cache miss events in CPU operation? • Cache hit: requested data found in cache → CPU accesses it quickly. • Cache miss: data not in cache → CPU fetches from slower main memory → slower access. • Significance: CPU performance depends heavily on cache efficiency; minimizing misses improves speed. 27. Describe the memory hierarchy model and arrange its levels in order of access speed. • Memory hierarchy: organizes storage from fastest/smallest to slowest/largest. • Levels (fastest → slowest): CPU registers → L1 cache → L2 cache → L3 cache → main RAM → secondary storage (SSD/HDD). • Purpose: keeps frequently used data close to CPU to minimize access time and maximize performance. 31. List the main components on a motherboard and describe the function of each. • CPU socket: holds the processor and provides electrical connections. • RAM slots: hold memory modules for temporary program/data storage. • Chipset: manages communication between CPU, memory, and peripherals. • Expansion slots (PCI/PCIe): connect graphics cards, network cards, and other peripherals. • Storage connectors (SATA, M.2): interface with HDDs/SSDs. • Power connectors: supply electricity to motherboard and components. • I/O ports: connect external devices (USB, HDMI, audio, network). • BIOS/UEFI chip: stores firmware for booting and hardware initialization. 32. Explain the three main types of buses: data, address, and control. • Data bus: transfers actual data between CPU, memory, and I/O. • Address bus: carries memory or I/O addresses from CPU to indicate where data is to be read/written. • Control bus: carries control signals (read/write, clock, interrupts) to coordinate operations. • Summary: data bus = content, address bus = location, control bus = operation signals. 33. Differentiate between serial and parallel ports. • Serial port: sends data 1 bit at a time; slower but simpler wiring; can transmit over long distances; used in USB, RS-232. • Parallel port: sends multiple bits simultaneously; faster for short distances; more pins needed; used for older printers and internal buses. • Key difference: data transmission method and distance/speed tradeoff. 34. Explain the key difference between PCI and USB interfaces. • PCI (Peripheral Component Interconnect): internal bus for expansion cards (graphics, network); parallel communication; high-speed; fixed internal connection. • USB (Universal Serial Bus): external interface for peripherals (keyboard, mouse, storage); serial communication; hot-pluggable; supports multiple devices. • Summary: PCI = internal, parallel, high-performance; USB = external, serial, versatile. 35. List the main differences between Von Neumann and Harvard architectures. • Von Neumann: single memory for instructions and data; simpler design; potential bottleneck due to shared bus. • Harvard: separate memories for instructions and data; allows simultaneous access; faster execution but more complex design. • Key difference: memory separation and access method. 36. What does a processor’s Instruction Set Architecture (ISA) define? • Defines the set of supported instructions, data types, registers, addressing modes, and hardware interface. • Determines how software communicates with hardware and what programs can run on the CPU. • ISA is the interface between software and processor design. 37. Describe the different addressing modes (Immediate, Direct, Indirect, Indexed, Register). • Immediate: operand is part of instruction (e.g., MOV AX, 5). • Direct: instruction specifies memory address of operand (e.g., MOV AX, [1000H]). • Indirect: memory address stored in register; operand fetched from that address (e.g., MOV AX, [BX]). • Indexed: base address + offset/index used to locate operand (e.g., MOV AX, [BX+SI]). • Register: operand is in CPU register (e.g., MOV AX, BX). • Summary: addressing mode determines how the CPU finds and accesses data. 38. What is the function of the Control Unit (CU), and how does it interact with other CPU components? • CU fetches instructions from memory, decodes them, and generates control signals. • Directs ALU for arithmetic/logic operations, registers for storage, and memory/I-O for data transfer. • Coordinates instruction sequencing, timing, and ensures proper execution of program flow. 31. List the main components on a motherboard and describe the function of each. • CPU socket: holds the processor. • RAM slots: hold memory modules. • Chipset: manages communication between CPU, RAM, and peripherals. • Expansion slots (PCI/PCIe): add graphics, sound, network cards. • Storage connectors (SATA, M.2): connect HDDs/SSDs. • Power connectors: supply electricity. • I/O ports: connect external devices. 32. Explain the three main types of buses: data, address, and control. • Data bus: carries data between CPU, memory, and peripherals. • Address bus: carries addresses specifying memory locations. • Control bus: carries control signals (read/write, interrupt, clock) to coordinate operations. 33. Differentiate between serial and parallel ports. • Serial port: sends data one bit at a time; slower but simpler, longer distance possible. • Parallel port: sends multiple bits simultaneously; faster, short distance, more pins needed. 34. Explain the key difference between PCI and USB interfaces. • PCI (Peripheral Component Interconnect): internal bus for connecting expansion cards (graphics, network); high-speed, parallel communication; stays inside PC. • USB (Universal Serial Bus): external interface for peripherals (keyboard, mouse, storage); serial communication; hot-pluggable and supports multiple devices. • Key difference: PCI is internal, parallel, high-performance; USB is external, serial, versatile for peripheral connectivity. 35. List the main differences between Von Neumann and Harvard architectures. • Von Neumann: single memory for data and instructions; simpler design; bottleneck possible (same bus for instructions and data). • Harvard: separate memories for data and instructions; allows simultaneous access; more complex but faster execution. • Key difference: memory separation and access method. 36. What does a processor’s Instruction Set Architecture (ISA) define? • Defines the CPU’s supported instructions, data types, registers, addressing modes, and hardware interface. • Determines what programs can run on that CPU and how they communicate with hardware. 37. Describe the different addressing modes (Immediate, Direct, Indirect, Indexed, Register). • Immediate: operand is part of instruction (e.g., MOV AX, 5) • Direct: instruction specifies memory address (e.g., MOV AX, [1000H]) • Indirect: memory address stored in register; operand fetched from that address (e.g., MOV AX, [BX]) • Indexed: base address + offset (index) used to locate operand (e.g., MOV AX, [BX+SI]) • Register: operand in CPU register (e.g., MOV AX, BX) 38. What is the function of the Control Unit (CU), and how does it interact with other CPU components? • CU fetches, decodes, and coordinates execution of instructions. • Sends control signals to ALU, registers, memory, and I/O devices. • Ensures proper sequencing, data flow, and timing within CPU.
